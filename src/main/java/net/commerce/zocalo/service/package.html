Classes involved in setting up a web service. {@link net.commerce.zocalo.service.AllMarkets} is the main entry point for a prediction market server. {@link net.commerce.zocalo.service.MarketOwner} helps by managing markets and users. {@link net.commerce.zocalo.service.CometServer} is an abstract superclass for servers that use AJAX.

{@link net.commerce.zocalo.service.ServerUtil} is an abstract superclass that reads info from config files. {@link net.commerce.zocalo.service.PropertyHelper} controls specific properties used in configuring Zocalo. {@link net.commerce.zocalo.service.Config} sets up configuration and manages access to some configured properties.

{@link net.commerce.zocalo.service.ServletUtil} is an abstract superclass that sets up JSP and other servlets under Jetty.

{@link net.commerce.zocalo.service.BayeuxSingleton} holds onto AJAX connections
